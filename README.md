# 🚀 ミッション：AIとレスキュー！魔法のウェブモンスター召喚カード

本日のミッションは，クラウド開発環境（GitHub Codespaces）を起動し，生成AI（Copilot等）と協力して，バグだらけでデザインが崩れた「モンスター召喚カード」を修復（レスキュー）することです．
完成したら，AIとの対話ログをJSONファイルとしてエクスポートし，すべてのファイルの変更履歴をセーブ（コミット）して，提出（プッシュ）してください．

---

## 📝 課題要件

### 1．モンスター画像の巨大化バグをレスキューする
初期状態では，モンスターの画像が巨大化（幅1200px）しており，画面からはみ出してしまっています．
`style.css` の `.card-image` の設定をAIに相談し，カードの幅にピッタリ収まるように修正してください．

### 2．ステータスを横並びにする（構造化と情報デザイン）
モンスターのステータス（属性，攻撃，防御）を示すリストが，縦に並んでしまっています．
AIに「Flexbox（フレックスボックス）」を使って，リスト項目をスッキリと横並びにする方法を相談し，`style.css` に適用してください．

### 3．カード全体のレイアウトと「召喚ボタン」を仕上げる
* カード全体（`.card-container`）の幅を制限し，画面の中央に綺麗に配置されるよう，影（シャドウ）や角丸などを追加してカードらしく整えてください．
* 地味で動かない「召喚ボタン（`.summon-button`）」を，AIのアドバイスをもとに魅力的な色や形（角丸など）にデザインしてください．
* さらに，ボタンにマウスをのせたときに，ふわっと上に持ち上がるような「ホバー効果（`:hover`）とアニメーション」を適用してください．

---

## 🤖 生成AI（Copilot）の活用ルール

本日の演習では，AI（Copilot Chat）をコーディングの相棒として使います．ただし，以下のルールを必ず守ってください．

1. **まずAIに聞いてみましょう：** 今回は，AIに直接CSSのコードを教えてもらい，それを貼り付けて動かす楽しさを体験します．
2. **仕組みを聞いて理解を深めましょう（能動的ヘルプシーキング）：** コードを貼り付けてプレビューで変化を確認したら，AIの返答の最後にある「次のステップのための質問例」から，気になるものを1つ選んでAIに送信し，コードの仕組みについて質問してください．
3. **対話は2往復以上行うこと：** 最低でも2回以上，AIに質問・相談を投げてください．

### 💡 AIへの質問プロンプトの例
* 「`.card-image` というクラス名の画像が，枠線からはみ出して巨大表示されてしまいます．画像の幅をカードの枠内にきれいに収めるためのCSSを教えてください．」
* 「`.status-list` というリストの項目を，Flexboxを使って横並びにするCSSを教えてください．」
* 「ボタンにマウスをのせたときに，ふわっと上に持ち上がり，背景色が変化するようなCSSのホバー効果とアニメーションを教えてください．」

## 🛠️ 操作手順（Step-by-Step）

### Step 1：リポジトリの作成とCodespacesの起動
1. Google Classroomに提示された課題のリンクをクリックし，アサインメントを受け入れます．
2. 生成された自分専用のリポジトリページを開き，緑色の **`Code`** ボタン ➔ **`Codespaces`** タブ ➔ **`Create codespace on main`** をクリックします．
3. ブラウザ上に開発画面（VS Code）が起動するまで待ちます（約1〜2分かかります）．

### Step 2：AIに相談してコードを修正する
1. 画面上のAIチャット（Copilot Chat）アイコンをクリックしてチャットパネルを開きます．
2. AIに最初の質問を投げます（上記のプロンプト例を参考にしてください）．
3. AIが提示したコードを `style.css` の指定された場所にコピー＆ペーストし，保存（`Ctrl + S` / `Cmd + S`）します．
4. プレビュー画面で，カードの表示がどのように変わったか確認します．
5. AIの回答の最後にある質問例を使って，AIに2回目の質問を投げ，仕組みを教わりましょう．
6. 同様に，ステータスリストの横並びや，ボタンのホバーアニメーションもAIと協力して完成させます．

### Step 3：AIとの対話ログをエクスポートする
1. AIチャットパネル of 右上にある **`…`（メニュー）** をクリックします．
2. 表示されたメニューから **`チャットのエクスポート（Export Chat）`** を選択します．
3. 保存場所をいま開いているワークスペース（リポジトリのフォルダ）に指定します．
4. ファイル名を **`copilot-chat-history.json`** としてそのまま保存します．

### Step 4：コミットとプッシュ（提出）
1. 左側メニューの **`ソース管理（ブランチのアイコン）`** を開きます．
2. 変更のあったファイル（`style.css`，`copilot-chat-history.json`）の横にある **`＋`** ボタンをクリックして「ステージ」します．
3. 上部のメッセージ入力欄に **`fix card layout and add hover effect`** と入力します．
4. **`コミット（チェックマーク）`** ボタンをクリックします（セーブ完了）．
5. 青色の **`変更の同期`** （またはプッシュ）ボタンをクリックします（送信完了）．

### Step 5：提出の確認
1. ブラウザで元のGitHubリポジトリページに戻り，リロードします．
2. ファイルリストに `copilot-chat-history.json` が追加されており，コミット履歴に「fix card layout and add hover effect」が記録されていることを確認します．
3. 確認できたら，そのリポジトリのURLをGoogle Classroomの提出欄に貼り付けて，提出を完了させてください．
